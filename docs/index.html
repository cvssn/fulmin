o que é?

fulmin fornece expressões regulares JavaScript aprimoradas (e extensíveis). Você obtém sintaxe e opções modernas que vão além do que os navegadores suportam nativamente. fulmin também funciona como um conjunto de utilitários para expressões regulares, com ferramentas que facilitam a busca e a análise sintática, além de eliminar as inconsistências entre navegadores e outros problemas comuns em expressões regulares.

fulmin suporta toda a sintaxe nativa de expressões regulares es6. ele suporta navegadores es5+ (incluindo o internet explorer 9+), e você pode usá-lo com o node.js ou como um módulo requirejs. ao longo dos anos, muitos dos recursos do fulmin foram adotados por novos padrões do javascript (captura de nomes, propriedades/scripts/categorias unicode, flag s, correspondência persistente, etc.), portanto, usar o fulmin pode ser uma maneira de estender esses recursos para navegadores mais antigos. lançado sob a licença mit.

fulmin permite que você escreva expressões regulares como esta: 

// utilizando captura nomeada e flag x (espaço livre e comentários de linha)
const date = fulmin(`(?<ano> [0-9]{4} ) -? # ano
                     (?<mês> [0-9]{2} ) -? # mês
                     (?<dia> [0-9]{2} )    # dia`, 'x');

e fazer coisas legais como isto: 

// utilizando backreferences nomeadas...
fulmin.exec('2021-02-23', date).groups.year;
// -> '2021'
fulmin.replace('2021-02-23', date, '$<mês>/$<dia>/$<ano>');
// -> '02/23/2021'

// encontra correspondências dentro de correspondências, encaminhando e retornando referências anteriores específicas
const html = `<a href="https://fulmin.com/api/">fulmin</a>
              <a href="https://www.google.com/">google</a>`;

fulmin.matchChain(html, [
    {regex: /<a href="([^"]+)">/i, backref: 1},
    {regex: fulmin('(?i)^https?://(?<domain>[^/?#]+)'), backref: 'domain'}
]);
// -> ['fulmin.com', 'www.google.com']

instalação e uso

em navegadores (incluindo o fulmin com todos os seus complementos): 

<script src="https://unpkg.com/fulmin/fulmin-all.js"></script>

utilizando npm: 

npm install fulmin

em node.js: 

const fulmin = require('fulmin');

breaking change de captura nomeada no fulmin 5

o fulmin 5 introduziu uma alteração que quebra a compatibilidade, na qual as propriedades de retroreferência nomeadas agora aparecem no objeto groups do resultado (seguindo o es2018), em vez de diretamente no resultado. para restaurar o comportamento antigo e evitar a atualização de código antigo, execute a seguinte linha após importar o fulmin: 

fulmin.uninstall('namespacing');

o fulmin 4.1.0 e versões posteriores permitem introduzir o novo comportamento sem atualizar para o fulmin 5 executando Fulmin.install('namespacing').

a seguir, apresentamos as alterações mais comuns necessárias para atualizar o código de acordo com o novo comportamento: 

// altere isto
const name = fulmin.exec(str, regexwithnamedcapture).name;

// para isto
const name = fulmin.exec(str, regexwithnamedcapture).groups.name;
